ROYECTO FINAL – ANÁLISIS   DE SISTEMAS 1 
Sistema de Votaciones 
 
Objetivo del proyecto 
Desarrollar la propuesta y una implementación básica de un Sistema de Votaciones, 
aplicando conceptos de planificación, metodologías ágiles (Scrum), análisis 
mediante historias de usuario, diseño del sistema, control de versiones y despliegue. 
Descripción del problema 
Actualmente, los procesos de votación se realizan de forma manual, lo cual genera 
problemas como: 
• Errores en conteo de votos  
• Retrasos en la consolidación de resultados  
• Falta de control en mesas electorales  
• Dificultad para auditar resultados  
Se requiere diseñar e implementar un sistema que permita gestionar el proceso de 
votación de manera ordenada, rápida y confiable. 
Trabajo en equipo 
• Grupos de 4 integrantes (no hay individuales)  
• Cada integrante debe asumir un rol específico  
• Todos Deben presentar  
• Evaluación individual y grupal 
Roles  
1. Project Manager  
2. Analista  
3. Desarrollador  
4. QA / Tester 
 
Metodología obligatoria: Scrum 
El equipo deberá aplicar Scrum de forma básica: 
Entregables Scrum: 
• Product Backlog (historias de usuario)  
• Sprint Backlog  
• Utilizar herramienta (atalasia, wrike,jira etc) 
• Definición de al menos 2 sprints 
 
Historias de usuario (mínimo requerido) 
El sistema debe incluir al menos: 
• Login  
• Crear usuarios  
• Crear votaciones  
• Crear mesas  
• Registrar resultados  
• Consultar resultados 
Formato obligatorio: 
Como [usuario] 
quiero [funcionalidad] 
para [beneficio] 
Costos 
El equipo debe estimar: 
• Costo de desarrollo (horas/persona)  
• Infraestructura (servidores, nube, etc.)  
• Licencias (si aplica)  
• Costo total del proyecto  
Diseño del sistema 
Debe incluir: 
• Diseño de módulos  
• Modelo de base de datos  
• Prototipos de interfaz 
 
 
Desarrollo 
Implementación mínima funcional de: 
• Login  
• CRUD de usuarios  
• Gestión de votaciones  
• Gestión de mesas  
• Registro de resultados 
Control de versiones (SCM) 
Uso obligatorio de repositorio en GitHub: 
• Commits por integrante  
• Uso de branches  
• Historial de cambios 
Pruebas 
• Definir pruebas basadas en historias de usuario  
• Validar funcionamiento del sistema
